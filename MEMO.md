docs/ 에 래더 자동 생성 R&D 3파일: `ladder_ir.py` (IR v0 조합논리) → `ladder_sim.py` (v1 SET/RST + TON 타이머 + 다중 스캔 simulate) → `ladder_search.py` (Spec/Scenario + score=trace 일치율−크기 페널티 + 무작위 탐색 베이스라인).

**현황 (2026-06-10):** 자기유지 스펙은 무작위 탐색 16k~43k 시도로 완벽해 발견 (M0 없이 Y0 직접 피드백 최소형). 꼼수 회로 `Y0=X0` 가 0.70 부분 점수 고원 형성 — 구조적 도약(피드백 도입) 필요 지점이 MCTS+정책망 도입 근거.

**순수 UCT 실증 → 벤치마크로 반증 (2026-06-10):** `docs/ladder_mcts.py` (스택 액션 PUSH/AND/OR/TON/PLS/EMIT/DONE + mean/max 혼합 UCB). 최초 "MCTS 4~40배" 결론은 **판정 기준 버그의 산물** — 옛 기준 `score≥0.99` 는 정답+크기≤10 을 동시 요구해서 크기 페널티로 수렴하는 MCTS 에 유리했음. 공정 기준 (accuracy==1.0) 의 `docs/ladder_benchmark.py` 6과제 사다리에서 **순수 UCT 는 무작위에 전패**: delayed_off/flicker 는 무작위 5k~46k 성공 vs MCTS 3시드 전패 (0.75~0.875 고원 고착). 원인: (a) 부분점수 높은 기만적 가지 (Y0=X0 류) 에 UCT 예산 집중, (b) 롤아웃 분포가 재귀 생성기보다 나쁨 (legal actions 의 PUSH 지배), (c) 분기 14~25 × 깊이 10~18 에서 200k 으로는 트리가 얕은 prefix 만 기억 — credit assignment 실패.

**스펙 작성 형식 논의:** 타임차트 (스캔별 입력/기대출력) + 부정/재진입/경계 시나리오로 꼼수 차단. 확장 필요 2건: don't-care (`?` 스캔 채점 제외, score() 한 줄) + invariant (전역 속성, fuzzing 검사 + 위반 시 큰 페널티 → CEGIS 연결). 제품화 방향: 타임차트 에디터 UI → Scenario 컴파일, 디바이스 목록은 CM/Action 정의에서 자동.

**벤치마크 결과 (2026-06-10, `ladder_benchmark.py` 6과제×무작위/MCTS×3시드, 예산 200k):** self_hold 무작위 7~3.4k / one_shot 양쪽 수십 회 / delayed_off·flicker 무작위만 성공 / **interlock·seq2 양쪽 전패**. interlock 3시드 동일 0.940 고원 = 탐색이 아니라 **스펙 과잉명세** — 기대 trace 를 레퍼런스 simulate 로 도출하면 1스캔 지연 같은 구현 부산물까지 스펙에 박혀 의도 동일·타이밍 1스캔 차이 대안 회로가 전부 감점. don't-care 필요성의 실측 증거. seq2 (크기 14) 는 진짜 탐색 한계로 추정. IR v2 에 Pulse(PLS) 추가 완료, accuracy()/score() 분리 완료 (accuracy 가 완벽해 판정 기준), BuildState 과제별 설정 가능 (timer_presets/allow_pulse/max_*).

**don't-care 실험 결착 (2026-06-11):** 파일은 `docs/reinforced_learning_mcts/` 로 이동된 상태. `mask_transition_scans(spec, width=1)` (입력 전이 스캔 기대값 None 치환, intent assert 는 마스킹 전 raw spec 에서 실행) + `assert_spec_not_degenerate` (항상0/항상1 회로가 acc 1.0 되면 raise — 초기 seq2 가 정확히 이 함정: 시나리오가 전이 위주로 압축돼 61% 마스킹 → 빈 회로 통과. dwell 스캔 연장으로 36% 정상화). interlock/delayed_off/seq2 에 적용, one_shot 은 펄스가 전이 스캔 자체라 제외. `ladder_ir.py` (v0) 삭제, `diag_plateau.py` 신설 (고원 회로의 실패 스캔 위치 + width=2 해소 가능 여부 표시).

**재측정 결과 — 과제별 원인이 셋으로 갈림:** ① **delayed_off = 과잉명세 실증**: 무작위 211~1,784회 발견 (마스킹 전 5k~46k, 약 25배 개선). ② **interlock = 꼼수 gradient 문제 → 탐색 한계로 재분류**: 0.940 고원의 정체는 `Y0=X0` (인터로크 없는 직결) 꼼수가 차단 의도 채점 1점만 희생하는 구조. **width=2 확대는 오답** — 그 1점이 마스킹돼 꼼수가 정답 판정됨. 시나리오 C dwell 연장 + 대칭 시나리오 D 추가로 꼼수 손실을 4점으로 키웠지만 여전히 0.933 고원 (6런 동일, 같은 아키타입) — 스펙은 건전해졌고 ref(크기 8, NC 교차결합 2 rung)를 200k 가 못 찾는 순수 탐색 한계. ③ **seq2 = 탐색 한계**: 0.910 고원 회로는 Y1 래치 부재 (X1 누른 동안만 ON), width=2 로도 정상 채점 스캔 3점 실패 잔존. MCTS 는 여전히 전 과제 무작위 이하 (delayed_off 0.833×3시드 동일).

**탐색 개선 1차 (2026-06-11 오후, 3종 구현+측정):** ① **가중 롤아웃 = MCTS 최초 성공.** `weighted_rollout` (KIND_WEIGHTS 종류 우선 샘플 — 균등 샘플의 PUSH 지배 10:2 해소) → delayed_off 2/3시드 발견 (336/5,191회, 기존 mcts 0/3). 벤치마크 3번째 방법 `mcts_w` 로 추가. ② **SET/RST EMIT 액션** (BuildState `allow_setrst` + GenCfg `setrst_p`, seq2 는 rung 한도 4 — SET/RST 래치 체인 자연형이 4 rung) → seq2 mcts/mcts_w 0.974 도달 (무작위 최고 0.910 추월), 발견은 실패. ③ **invariant 는 interlock 을 못 풀었음 (가설 반증).** `Spec.invariants` (스캔별 출력 전역 속성, 마스킹 스캔에도 적용) + `evaluate()` (acc/viol 동시 산출, simulate 1회) + score 위반당 -0.05 + 완벽해 판정 `acc==1.0 and viol==0`. 결과 9런 전부 0.933 — invariant 는 트리 탐색의 보상 지형만 바꾸고 **무작위 샘플링에는 원리적으로 무영향** (독립 시도라 gradient 없음), MCTS 도 꼼수 분지 억제만으로는 크기 8 교차결합 2 rung 을 조립 못 함. interlock 의 본질 난점은 "두 rung 동시 정합" — 한 후보의 좋은 Y0 rung 과 다른 후보의 좋은 Y1 rung 을 합치는 **rung 단위 교차(GP)의 교과서적 사례**. `smoke_test.py` 신설 (spec 가드 + EMIT ops + 3방법 무크래시, 수 초).

**GP 도입 — 난이도 사다리 전 과제 결착 (2026-06-11 저녁):** `ladder_gp.py` (rung 목록 단일점 교차 + 변이 5종 + 토너먼트 k=3 + 엘리트 적합도 캐시 + 새 피 5%, 교차 후 Timer/Pulse 이름 재부여로 부모 간 상태 공유 차단, 생성기는 benchmark 주입 재사용). **interlock 3/3 (1,148/5,087/14,643회), seq2 3/3 (16,890/91,151/116,550회)** — 다른 3방법 전패 과제를 GP 만 풂.
